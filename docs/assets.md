# LUCID v1.0 Asset Checklist

Put public-facing demo assets in these folders. Do not include private names, phone numbers, account numbers, faces, addresses, QR codes, or real bank details unless fully blurred.

## Case Images

Folder: `public/cases/`

Recommended files:

- `case-01-low-risk.png`  
  A normal reminder / harmless chat screenshot.
- `case-02-airline-refund.png`  
  A desensitized airline refund phishing screenshot.
- `case-03-embassy-police.png`  
  The main fake embassy / police scam screenshot.

## Product Screenshots

Folder: `public/screenshots/`

Recommended files:

- `console-red-bodyguard.png`  
  LUCID console showing Red risk and Bodyguard Mode.
- `console-upload-analysis.png`  
  Screenshot upload with evidence view.
- `pattern-memory-mongodb.png`  
  MongoDB Pattern Memory proof or app result.

## Brand / Video Assets

Folder: `public/brand/`

Recommended files:

- `hero-background.jpg`  
  Optional dark security-style background image.
- `demo-cover.png`  
  Thumbnail for Devpost / video cover.

## Image Rules

- Use PNG or JPG.
- Keep each file under 2 MB when possible.
- Blur or crop all personal identifiers.
- For Chinese screenshots, keep text sharp and avoid over-blurring the message bubbles.
- Main demo image should show a complete manipulation chain: authority, fear, isolation, urgency, and financial demand.
